# 选题分析 Workflow

**两种触发模式**：
- **模式 A（推荐）**：仅输入 `/01-topic-research` → AI 批量生成 15-20 个候选选题并打分 → 人工选择 → 进入内容策划环节。
- **模式 B**：输入 `/01-topic-research 具体选题描述` → 跳过选题池，针对该选题生成 **20 个候选标题** → 人工选择 → 进入内容策划环节。

---

## 前置依赖

执行前自动读取：
- `Project Context (项目上下文).md`：核心定位、内容方法论
- `Content Plan (内容计划).md`：选题排序原则、内容规划
- `content-library/PIPELINE.md`：往期选题状态与标题
- `ideas/backlog.md`：候选选题池

## 流程分支

### 分支 A：用户未提供具体选题（推荐路径）

#### 阶段 1：批量选题池生成（AI 主导）

1. 读取上述所有前置依赖，提取频道母题、已做选题、内容规划策略。
2. 调用 `topic-analyst(选题分析师).md` 的【批量选题生成 Prompt】。
3. 基于往期选题和频道母题，生成 **15-20 个候选选题**。
4. 每个选题按以下维度打分（0-100），并以 Markdown 表格展示：

| 维度 | 说明 |
|------|------|
| **母题匹配度** | 与"AI IDE 真实场景能力验证"的契合程度 |
| **实操难度** | 完成 Demo 所需的时间和复杂度（分数越高越简单） |
| **痛点强度** | 目标受众面临该问题的普遍性和紧迫性 |
| **差异化** | 与现有中文 AI IDE 内容的差异化程度 |
| **流量潜力** | SEO 搜索量 + 社媒传播性的综合预估 |
| **综合得分** | 加权计算：匹配度×0.25 + 痛点×0.25 + 差异化×0.2 + 流量×0.2 + 难度×0.1 |

5. **输出指令**：展示完整列表后，提示用户：
   > "请从以上选题中回复**序号**选择 1 个，或提供你自己的新选题描述。选定后，将进入内容策划环节。"

#### 阶段 2：人工选择

- 用户回复序号或新选题。
- 记录用户选择，进入 `02-content-planning (内容策划)` 环节。

### 分支 B：用户已提供具体选题

跳过阶段 1-2。

#### 阶段 2B：标题池生成（针对单选题）

1. 基于用户提供的选题方向，按"工具名 + 时效钩子 + 反预期 + 数字/对比"四要素，发散生成 **20 个候选标题**。**反噱头准则**：「数字/对比」可用，但卖点不得建立在代码行数（如“100 行”）等与价值无关的噱头上（AI 编程时代代码长度不敏感），优先用“工程结构/可复现性/真实能力边界”做钩子。
2. 每个标题标注钩子类型（如：数字对比、反预期、否定式、方法论承诺等）。
3. 以 Markdown 表格展示，等待用户回复序号选择 1 个标题。

#### 阶段 3：人工选择标题

- 用户回复序号或提供自定义标题。
- 记录选择，进入 `02-content-planning (内容策划)` 环节。

## 使用示例

**示例 1：批量生成选题池（推荐）**
```text
/01-topic-research
```
预期：AI 先生成 15-20 个候选选题打分列表，等待用户选择。

**示例 2：直接分析具体选题**
```text
/01-topic-research
候选选题：Cursor Rules .mdc 机制深度解析
```
预期：AI 针对该选题生成 20 个候选标题供选择，选定后进入内容策划。
