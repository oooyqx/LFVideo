# 分发适配 Workflow (12-distribute-adapt)

基于上游定稿的**质检通过成片（11）**，调用 `分发助手` 角色，完成多平台（小红书、知乎、抖音、B站、YouTube 等）的文案适配、发布标题、话题标签设计与短视频切片策略。

---

## 前置依赖

本工作流假设已完成 `/11-qa-review`，已具备：
- 处于 `approved` 状态的 `content-library/<epNN-slug>/11-qa/README.md`
- 质检通过的 MP4 视频成片（横版/竖版）、字幕文件、封面图

如果缺少上述输入，先提示用户回到对应上游阶段。

---

## 步骤

### 1. 加载角色定义

读取 `shared/roles/content/distributor(分发助手).md`（若无，加载默认宣发分发大纲），理解其：
- 各大主流平台（小红书、知乎、抖音、B站、YouTube）的受众属性
- 平台推荐逻辑与文风偏好

### 2. 确认输入基础信息

从 `02-plan/README.md` 和 `04-script/README.md` 提取：
- **定稿标题**
- **核心痛点与避坑 Rule（判断层）**

### 3. 生成多平台适配文案

严格遵循分发规则，生成各平台的具体发布物：

**小红书**
- 标题：痛点型 / 结果型钩子
- 正文：短文案，引导看片
- 图片：6-9 张卡片（复用 04-script 产生的小红书文案卡片）
- 必含一张「边界/避坑指南」卡片（差异化）

**知乎**
- 标题：信息量优先，少标题党
- 文章框架：引子 → 操作层 → 判断层（边界与验收） → 结语
- 话题标签：3-5 个

**抖音**
- 短视频切片标题：钩子公式 `[反预期结果] + [制造悬念] + [行动指令]`
- 话题标签

**B站 / YouTube**
- 主视频发布标题（如：《2026年的Cursor和Windsurf，已经不是你以为的AI IDE了》）
- 简介描述栏（含配套开源 GitHub 链接、核心 timeline 高光打点）
- 话题标签/Tags

### 4. 短视频切片策略（需 9:16 竖版）

基于成片标注高光时间点，产出切片清单：
| 切片类型 | 时长 | 用途 |
|---------|------|------|
| 反预期开场 | 15-30秒 | 抖音 / 小红书 |
| 高光操作 | 30-60秒 | 抖音 / 小红书 |
| 边界/避坑瞬间 | 15-30秒 | 小红书 |
| 数据冲击 | 10-15秒 | 抖音 |

输出：每个切片的时间区间 + 配套 hook 文案

### 5. 自我检查（不输出，仅约束）

- 标题是否同时具备 SEO + 钩子，且不标题党超标？
- 是否擅自改动了核心观点或边界验收立场？（禁止）
- 各平台标签是否匹配该平台算法？
- 短视频切片是否真正是"钩子产品"而非长视频简单截取？
